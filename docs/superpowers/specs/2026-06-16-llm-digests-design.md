# LLM Digests — design (APPROVED)

> **Status: approved (2026-06-16).** Решения приняты пользователем:
> доставка — **в интерфейс**; ключ — **переиспользуем существующий `LLM_API_KEY`**
> (новый ключ/зависимость не нужны); авто-расписание — **позже** (на старте ручной
> запуск + UI). Реализация разблокирована.
> Extends the story-timeline layer (см. `2026-06-15-story-timeline-design.md` §3.5, §4).
> Дата: 2026-06-16. Ветка: `feat/story-timeline`.

## 1. Цель

«Утренняя сводка» по топ-сюжетам за сутки: LLM (Claude Haiku 4.5) генерирует
короткий аналитический дайджест **по агрегатам** (story + story_points +
инциденты), не по сырым постам. Это последний пункт MVP-роадмапа из исходного
спека (§4, фаза 4). Модель Claude Haiku 4.5 выбрана при первом брейншторме.

## 2. Что уже есть (фундамент)

- `Story` / `StoryPoint` / `Incident` + `is_anomaly` — готово.
- `reports` таблицы пока НЕТ (в исходном спеке §9 она есть: `reports(story_id,
  kind, body, created_at)`) — её добавляем здесь.

## 3. Провайдер LLM и безопасность стоимости (ключевой принцип)

**Переиспользуем существующую интеграцию** из `drafts.py`: env `LLM_API_KEY` +
`LLM_API_URL` (Anthropic-совместимый прокси `aiprimetech.io`), вызов через `httpx`
с заголовками `x-api-key` + `anthropic-version: 2023-06-01`, payload
`{model, max_tokens, system, messages}`, модель `claude-haiku-4-5-20251001` (как в
`drafts.py`). **Новый ключ/SDK/зависимость не нужны.**

LLM-вызовы платные → слой строго **degrade-to-noop**:
- Без `LLM_API_KEY` — генерация не вызывается, эндпоинт отдаёт `503 «not configured»`
  (как уже делает draft-эндпоинт), планировщик ничего не гонит.
- Авто-генерация по расписанию — **вне старта** (добавим позже за флагом).
- Лог токенов (`usage` из ответа), мягкий потолок `max_tokens`.
- Тесты **мокают** HTTP-вызов — ни одного реального обращения к LLM в тестах.

## 4. Компоненты

- `backend/radar/llm.py` — тонкий хелпер `complete(system, user, max_tokens, model)
  -> str` поверх `httpx.post(LLM_API_URL, ...)` по образцу `drafts.py` (те же
  заголовки/формат). Если `LLM_API_KEY` пуст → `LLMNotConfigured`. Парсит
  `resp.json()["content"]` → текст. (DRY: digests и потенциально drafts могут
  опираться на него; drafts не трогаем в этой итерации.)
- `backend/radar/digests.py` — `build_daily_digest(session, brand_id) -> Report|None`:
  выбирает топ-N активных сюжетов за сутки (по объёму/аномальности), собирает
  компактный текстовый агрегат (заголовок, динамика, тональность, топ-источники,
  is_anomaly), один вызов `llm.generate`, сохраняет `Report(kind="digest")`.
- `Report` модель (`reports`): `id, brand_id, story_id?, kind, body, created_at`.
- API: `POST /brands/{id}/digest` (сгенерировать сейчас), `GET /brands/{id}/digests`
  (список) — оба через `_owned_brand`.
- Фронт (echo-app): экран/панель «Дайджесты» — список сгенерированных сводок.
- (Опц.) Планировщик: раз в сутки для брендов с включённым флагом.

## 5. Структура промпта (фикс. формат отчёта, по спеку §3.5)

`тема → динамика → ключевые источники → риски → рекомендованное действие`.
Системный промпт задаёт роль аналитика и русский язык; user-контент — агрегаты.
Кэширование промпта (system) по желанию (см. claude-api prompt-caching).

## 6. Фазы (когда одобрят)

1. `llm.py` + тесты (мок).
2. `Report` модель + миграция/таблица.
3. `digests.py` (`build_daily_digest`) + тесты (мок LLM, реальные агрегаты).
4. API эндпоинты + тесты (`_owned_brand`).
5. Фронт «Дайджесты».
6. (Опц.) планировщик за флагом.

## 7. Решения (ПРИНЯТЫ 2026-06-16)

1. **Канал доставки → в интерфейс.** Сохраняем `Report` + показываем на экране
   «Дайджесты» в echo-app. Telegram/email — не сейчас.
2. **Ключ → переиспользуем `LLM_API_KEY`** (уже есть в `backend/.env`). Новый ключ
   не нужен; degrade-to-noop без ключа.
3. **Авто-расписание → позже.** На старте только ручной эндпоинт + UI. Фаза 6
   (планировщик за флагом) — вне этой итерации.

## 8. Риски

- **Стоимость LLM — главная ловушка** (по спеку §5): поэтому строго по агрегатам,
  раз в сутки, opt-in, с логом токенов.
- Качество русских сводок Haiku — проверить на реальных брендах
  ([[echo-testing-real-brands]]); при нужде поднять до Opus 4.8.
