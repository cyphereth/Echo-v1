# Антиспам: стоп-слова + примеры (гибридный фильтр)

**Дата:** 2026-06-26 · ветка `feat/intel-thread-context`

## Проблема
В ленту попадают ненужные посты. Нужен механизм, куда куратор «скидывает»
мусор, а система запоминает его и отсекает похожее в будущем.

## Решение — гибрид
1. **Быстрый слой — стоп-слова:** детерминированный чёрный список слов/фраз
   (добавляются вручную). Пост со стоп-словом отбрасывается мгновенно, без LLM.
2. **ИИ-слой — примеры:** скинутые из ленты посты сохраняются как «примеры мусора».
   На каждом тике survivors одним батч-вызовом Claude сравниваются с примерами
   («отметь похожие»). Fail-open: нет ключа/ошибка → ничего не режем.

Стоп-слова заводятся руками; примеры копятся автоматически при скидывании из ленты.

## Модель данных — таблица `intel_spam`
Создаётся через `create_all()` (миграция не нужна — новая таблица).

| поле | тип | назначение |
|---|---|---|
| `id` | PK int | |
| `kind` | Text `"word"`\|`"example"` | стоп-слово или пример-пост |
| `value` | Text | слово/фраза или текст поста-примера |
| `author` | Text, null | автор примера |
| `source_post_id` | Text, null | post_id скинутого поста |
| `note` | Text, default "" | пометка |
| `created_at` | datetime | |

`UniqueConstraint(kind, value)` — дедуп.

## Слой фильтрации — `radar/intel/spam_filter.py`
- `load_spam(session) -> tuple[list[str], list[dict]]` — стоп-слова и примеры.
- `blocked_by_word(text, blocklist) -> bool` — по границе слова, регистронезависимо.
- `classify_spam_batch(texts, examples) -> list[bool]` — батч-вызов Claude (копия
  паттерна `core/spam.classify_ads_batch`), `True` = спам. Fail-open: `[False]*n`
  при отсутствии ключа/ошибке.

### Встраивание в `collector.collect_probe` (обе ветки)
1. Грузим `(blocklist, examples)` один раз за вызов рядом с `lexicon_terms`.
2. Быстрый слой: после admit-проверки пост со стоп-словом отбрасывается.
3. ИИ-слой: survivors буферизуются; в конце один `classify_spam_batch` с примерами —
   спам **не пишется в БД** (не засоряет stories/агрегаты, колонка `is_spam` не нужна).
4. Watermark двигается как раньше (фиксируется до фильтрации).

Если примеров нет или нет ключа — ИИ-слой пропускается, поведение как сейчас.

## API — `radar/intel/api.py`
- `GET /intel/spam?kind=word|example` — список (фильтр по kind опционален).
- `POST /intel/spam` — `{ kind, value, author?, source_post_id?, note? }`; валидация
  `kind ∈ {word, example}`, непустой `value`; дедуп (повтор → 200 с существующей).
- `DELETE /intel/spam/{id}` — удалить запись.

## Фронтенд
- **`echo-app/src/features/intel/api.js`:** `spamList(kind?)`, `addSpam(body)`,
  `deleteSpam(id)`.
- **Лента (Stream):** кнопка «Спам» на каждом посте → `addSpam({ kind:'example',
  value:text, author, source_post_id })`; пост оптимистично прячется из ленты.
- **Новая страница «Антиспам»** (`IntelSpam.jsx`): добавление/удаление стоп-слов +
  список примеров с удалением. Пункт в навигации intel.

## Тесты
- `spam_filter`: границы слов/регистр для `blocked_by_word`; fail-open для
  `classify_spam_batch` (нет ключа → все `False`).
- `collector`: пост со стоп-словом дропается; ИИ-слой замокан (пример → дроп похожего).

## Вне объёма
- Авто-извлечение стоп-слов из примеров (слова заводятся вручную).
- Эмбеддинги/косинусная близость (выбран батч-LLM вариант A).
- Ретро-чистка уже собранных мусорных упоминаний.
