# Echo Radar — Design System

> **Echo Radar** ловит вирусный негатив о бренде в Instagram и TikTok, пока у ролика
> или комментария тысяча просмотров, а не миллион — и сразу отдаёт готовый черновик
> ответа на одобрение. Вы тушите репутационный пожар, пока он управляемый.

Дизайн-система задаёт визуальный и тональный язык продукта: тёмный «диспетчерский
центр», где всё спокойно и под контролем, а срочность проявляется единственной
тёплой точкой — сигналом severity.

---

## 1. Контекст продукта

**Что это.** B2B-инструмент репутационного радара для брендов на рынке РФ/СНГ.
Продукт ловит публичные упоминания бренда в **Instagram и TikTok**, классифицирует их
по тону и виральности и отдаёт PR/SMM-менеджеру карточку с готовым черновиком ответа.

**Уникальность — не «ещё один мониторинг», а раннее предупреждение о виральности.**
Система предсказывает, *залетит* ли негативный ролик, пока он ещё не набрал охват.

**Эталонный сценарий.** Блогер жалуется на испорченный заказ → ролик начинает расти →
радар ловит это на разгоне → пуш в Telegram PR-менеджеру за минуты с черновиком ответа →
бренд тушит до того, как это стало кризисом.

**Кто покупатель.** PR + SMM (часто один отдел).

### Ключевые понятия продукта (словарь интерфейса)
| Термин | Значение |
|---|---|
| **Зонд (probe)** | Поисковый агент по ключевому слову или хэштегу с «водяной меткой» (watermark) |
| **Упоминание (mention)** | Уникальный пост/коммент, пойманный зондом |
| **Severity** | Скоринг виральности 0–100. Главное число интерфейса |
| **Opportunity** | Скоринг «позитивной» возможности (амбассадоры, удачный отзыв) |
| **Фаза (phase)** | `rising` (растёт) · `declining` (затухает) · `unknown` |
| **Тон (tone)** | negative · neutral · positive |
| **Категория** | viral_negative · complaint · neutral · positive · casual |
| **Полоса (lane)** | Маршрут карточки: **PR** (срочное, high-severity) или **SMM** |
| **Hot-watch** | Учащённый перезапрос растущих упоминаний для пересчёта Severity |
| **Confidence-гейт** | При низкой уверенности классификатора черновик НЕ предлагается → «решает человек» |
| **Черновик (draft)** | Сгенерированный ответ; всегда с конкретным следующим шагом. Решение за человеком |

### Продукты / поверхности
1. **Веб-приложение (инбокс / диспетчерский центр)** — основная рабочая поверхность.
   Две полосы PR/SMM, карточка упоминания, скоринг, график роста, черновик ответа,
   действия одобрить/править/отклонить, онбординг зондов. → `ui_kits/app/`
2. **Маркетинговый сайт (лендинг)** — объясняет ценность «раннего предупреждения». → `ui_kits/website/`
3. **Telegram-пуш** — срочный PR-алерт (ссылка + тон + Severity + черновик). Представлен
   как компонент внутри UI-кита приложения.

## 2. Источники

Эта система построена из двух материалов (стороннего кода UI / Figma **не было** —
бэкенд-репозиторий определяет домен, фронтенд спроектирован с нуля по бренд-брифу):

- **Бренд-бриф** (от заказчика): тёмный сине-графитовый фон, холодный циан как акцент,
  настроение «спокойный диспетчерский центр»; severity вспыхивает тёплым — алый
  «залетает», янтарный «растёт». Спокойствие как фон, срочность как точка.
- **`uploads/CLAUDE.md`** — инженерный контекст бэкенда Echo Radar: модель данных
  (`brands`, `probes`, `mentions`, `mention_snapshots`), пайплайн
  (Сбор → Дедуп → Классификация → Скоринг → Черновик → Доставка), скоринг
  (velocity/acceleration/phase/Severity/Opportunity), инварианты, фазы сборки P1–P7.
  Это источник истины для доменной логики и терминологии интерфейса.

> Готового фронтенд-кода или Figma-файла нет. Весь UI в `ui_kits/` — оригинальный
> дизайн, построенный на бренд-брифе и доменной модели из CLAUDE.md.

## 3. Content Fundamentals — как пишется копирайт

**Язык.** Интерфейс и копирайт — **на русском**. Технические идентификаторы (severity,
probe, lane, API-имена) могут оставаться английскими как термины.

**Адресация.** К пользователю — на **«вы»**, строчное, без подобострастия:
«Вы тушите пожар, пока он управляемый». О системе — от третьего лица или нейтрально
(«радар ловит», «продукт предлагает черновик»), без «я».

**Тон.** Спокойная компетентность диспетчера. Уверенно, по делу, без паники — даже
когда речь о кризисе. Срочность передаётся **данными и цветом**, а не восклицаниями.
Никаких «🔥 СРОЧНО!!!». Severity 87/100 и алая точка говорят сами за себя.

**Casing.** Заголовки — обычным предложением (sentence case), не Title Case и не КАПСОМ.
КАПС только для mono-лейблов/эйброу в интерфейсе («SEVERITY», «ФАЗА: РАСТЁТ»), где он
работает как приборная маркировка.

**Конкретика вместо воды.** Черновик ответа всегда содержит **следующий шаг**:
не «нам очень жаль», а «пришлите номер заказа в личные сообщения — проверим сегодня».
Это же правило в маркетинге: «за минуты», «пока тысяча просмотров, а не миллион»,
а не «быстро» и «эффективно».

**Логотип.** Вордмарк «Echo» с сигнальными волнами, исходящими из слова (продукт
«вещает» и слушает эфир); пойманный тёплый блип — в волнах. Квадратный знак для
апп-иконки — те же волны из нижне-левого источника. Файлы: `assets/logo-full-dark.svg`,
`assets/logo-mark.svg`.

**Метафора бренда.** Радар / диспетчерская / сигнал / разгон / тушим пожар. Глаголы
действия: ловит, предсказывает, тушит, разгоняется, залетает, растёт, затухает.

**Числа — округляем** для человека (1.2k просмотров, +340/час), точные значения — в mono.

**Эмодзи — нет.** В продукте эмодзи не используются как украшение интерфейса. Эмодзи
могут появляться только внутри *цитируемого* пользовательского контента (текст поста).

**Примеры голоса:**
- Hero: «Ловите негатив, пока у ролика тысяча просмотров, а не миллион».
- Подзаголовок: «Echo Radar предсказывает виральность на разгоне и отдаёт готовый
  черновик ответа — вы тушите пожар, пока он управляемый».
- Пустое состояние инбокса: «Тихо. Зонды на связи, новых сигналов нет».
- Severity-лейбл: «ЗАЛЕТАЕТ» (алый), «РАСТЁТ» (янтарный), «ПОД КОНТРОЛЕМ» (циан/зелёный).
- Действие: «Одобрить и отправить» · «Править» · «Отклонить».

## 4. Visual Foundations

**Общее настроение.** Диспетчерский центр ночью: тёмный сине-графитовый интерфейс,
приглушённый и собранный. Холодный циан — это «всё под контролем»: им подсвечены
радар, активные элементы, бренд. Severity — единственное тёплое пятно: алый и янтарный
вспыхивают только там, где есть срочность. **Спокойствие занимает 95% экрана, срочность —
одну точку.** Это главный закон системы: тёплый цвет — дефицитный ресурс.

**Цвет.**
- Фоны — холодный графит-навигатор: `--bg-void #0A1420` → `--bg-base #0F1E2E` →
  `--bg-raised #16293B`. Карточки на `--surface-1 #1C3349`, инпуты/raised — `--surface-2`.
- Бренд — холодный циан `--brand #2BB3C7`, ярче `--brand-bright #57D2E2`.
- Текст — прохладные белые: `--fg-1 #EAF1F8` → `--fg-2 #97A9BE` → `--fg-3 #5C6E83`.
- Severity — **тёплые**: критично `--sev-critical #FF4D5E` (алый, «залетает»),
  растёт `--sev-rising #FFB23E` (янтарный), под контролем `--sev-calm #34D8A0`
  (холодный зелёный, изредка). Тёплое НИКОГДА не используется декоративно.
- Тон-семантика: негатив = алый, нейтрал = `#7E91A6`, позитив = зелёный.

**Типографика.** Manrope (UI, заголовки, тело) + JetBrains Mono (данные, метрики,
severity, таймстемпы, эйброу). Заголовки — плотные (700–800), отрицательный трекинг.
Числа и приборные лейблы — всегда mono, tabular-nums, КАПС с разрядкой 0.16em для
эйброу. Mono несёт «инструментальный» голос; sans — человеческий.

**Spacing.** Сетка 4px (`--s-1`…`--s-20`). Интерфейс плотный, но дышащий: карточки
с внутренним отступом 16–24px, секции лендинга 64–80px. Данные группируются плотно,
заголовки получают воздух.

**Фоны / текстуры.** Преимущественно плоский графит. Допустимый мотив — **очень
деликатная радарная сетка**: концентрические окружности и/или тонкая координатная
решётка с opacity 0.04–0.10 на циане. Никаких фотофонов в продукте; на лендинге —
допустим тёмный градиентный «scope» (радиальное свечение циана от центра) за hero.
Полноцветных пёстрых градиентов и сине-фиолетовых переливов — НЕТ.

**Свечения (glow).** Ключевой приём вместо ярких заливок. Спокойная пульсация циана
(`--glow-brand`) на активных/радарных элементах; тёплая вспышка (`--glow-critical`,
`--glow-rising`) — только на high-severity сигналах. Glow = свет прибора, а не
неоновая реклама: мягкий, низкой непрозрачности.

**Анимация.** Сдержанная и функциональная. `--ease-out` для появлений, 120–220ms.
Радарный «sweep» (медленное вращение луча, 4–8s linear) и мягкий «pulse» severity-точки —
единственные продолжительные анимации. Никаких bounce и пружин: диспетчерская не
подпрыгивает. Числа Severity могут «докручиваться» (count-up) при появлении карточки.

**Состояния.**
- Hover: подъём поверхности на один шаг (`surface-1` → `surface-2`), бордер
  `line-2` → `line-strong`, у интерактивных — лёгкий cyan-glow. Без резкой смены цвета.
- Press/active: затемнение на ~6% + transform: scale(0.985). Быстро (120ms).
- Focus: 2px кольцо `--brand-line` со смещением 2px (видимое, доступное).
- Selected: левый/нижний бордер-акцент циан + фон `--brand-ghost`.

**Бордеры.** Тонкие холодные хайрлайны: `--line-1` (разделители), `--line-2` (карточки),
`--line-strong` (акцент/hover). 1px. Severity-карточки получают цветной бордер-намёк
(`--sev-*-line`) слева или по периметру.

**Тени.** На тёмном тени мягкие и глубокие, не «парящие»: `--shadow-1/2/3`. Чаще
elevation читается через свет (поверхность светлее) и бордер, а не через тень.

**Скругления.** Умеренные, техничные: карточки `--r-md 10px` / `--r-lg 14px`, кнопки
`--r-md`, пилюли-бейджи `--r-pill`, инпуты `--r-md`. Не «пухло»: радиус сдержанный.

**Карточки.** `--surface-1`, бордер `--line-2`, радиус `--r-lg`, `--shadow-2`.
Severity-карточка добавляет тёплый акцент (полоса/glow) пропорционально остроте.
Внутри — структура: шапка (платформа + время + severity), тело (текст упоминания),
подвал (черновик + действия).

**Прозрачность и блюр.** Используются точечно: «протекторные» оверлеи (затемнение
под выпадашками/модалками `rgba(7,11,17,0.6)` + `backdrop-blur`), стеклянный
top-bar при скролле. Не как украшение.

**Сетка / фиксированные элементы.** Приложение: фиксированный левый сайдбар (нав) +
фиксированный top-bar, скроллится центральная колонка инбокса; справа — детальная
панель карточки. Лендинг: центрированный контейнер max-width ~1120px, фиксированный
прозрачно-стеклянный хедер.

## 5. Iconography

**Система.** [Lucide](https://lucide.dev) — тонкие линейные иконки (stroke 1.5–2px,
без заливок), которые читаются как приборные пиктограммы диспетчерской. Подключается
с CDN (см. `ui_kits/*/index.html`); собственного иконочного шрифта в продукте нет.

**Почему Lucide.** Открытый, лёгкий, единый stroke-стиль, отлично смотрится тонкими
циановыми линиями на тёмном. Если потребуется офлайн — ставится npm-пакетом `lucide`.

**Ключевые иконки и их роли:**
- `radar`, `radio`, `satellite-dish` — бренд/радар/зонды
- `activity`, `trending-up`, `flame` — рост, виральность, «залетает»
- `bell`, `send`, `message-square` — алерты, доставка, Telegram-пуш
- `shield-check`, `circle-check`, `check` — под контролем, одобрить
- `pencil`, `x`, `eye` — править, отклонить, «решает человек»
- `instagram` — Instagram; для **TikTok / Twitter(X) / Telegram** в Lucide единых
  фирменных глифов нет → используем встроенные SVG-знаки платформ (см.
  `preview/brand-platform-marks.html`). Площадки v1 — Instagram + TikTok; Twitter и
  Telegram размечены как «скоро» (коннекторы v2).

**Эмодзи / Unicode-иконки.** Не используются в хроме интерфейса. Допустимы только
внутри цитируемого пользовательского контента (текст пойманного поста).

**Платформенные марки.** Instagram — `--pf-instagram #E1306C` (только сам глиф),
TikTok — белый `--pf-tiktok`. Используются мелко, как метки источника на карточке.

## 8. Caveats

- **Шрифты** (Manrope, JetBrains Mono) грузятся с Google Fonts CDN, не бутстрапятся
  локально. Если нужен офлайн-пакет `.woff2` — попросите, добавлю в `fonts/`.
- **Логотип и радарный знак** созданы с нуля под бренд-бриф (готового лого не было) —
  это предложение, а не утверждённый фирменный знак. Открыт к итерации.
- **Глифы площадок** — встроенные SVG-знаки (см. флаги выше).
- UI-киты — оригинальный дизайн по брифу + доменной модели; не реверс существующего
  фронтенда (его не было).
